# backsteros-mobile

Expo SDK 54 + Expo Router product client (Phase 4).

**Status:** Clerk sign-in, Inbox + Journal + Tasks + Projects with PowerSync and
offline status edits. Falls back to REST when sync is unavailable. Document
body open-on-demand is next (journal tab is a title list for now).

Tab chrome is two floating blurred pills (`components/floating-tab-bar.tsx` +
`expo-blur`, hosted in `FullWindowOverlay` on iOS): main nav Inbox ‚Üí Journal ‚Üí
Tasks ‚Üí Projects, and a separate compose pill (desktop create icon).

## SQLite adapters

| Environment | Adapter |
| --- | --- |
| Expo Go (`executionEnvironment === "storeClient"`) | `@powersync/adapter-sql-js` (JS only) |
| Dev client / production (`expo run:*`, EAS) | `@powersync/op-sqlite` + `@op-engineering/op-sqlite` (New Arch) |

```bash
# Expo Go (SQL.js) ‚Äî limited; prefer native for foundation work
pnpm --filter @backsteros/mobile start

# Native SQLite (requires Xcode once)
pnpm --filter @backsteros/mobile prebuild
pnpm --filter @backsteros/mobile ios
```

`ios/` and `android/` are generated by `prebuild` (gitignored). Re-run `prebuild`
after adding Expo native modules (e.g. `expo-web-browser`).

## Develop

```bash
cp backsteros-mobile/.env.example backsteros-mobile/.env
# Set EXPO_PUBLIC_CLERK_PUBLISHABLE_KEY (+ LAN API URL for devices)
pnpm install
pnpm --filter @backsteros/mobile ios
```

Requires a running API. PowerSync (`GET /api/v1/powersync/token`) enables local
SQLite; without it, screens use REST fallback.

No demo fixtures when signed out or when the Clerk key is missing.

## Production / release

- **EAS** ‚Äî `eas.json` profiles: `development`, `preview`, `production`. Run from
  this package with the EAS CLI (`eas build --profile preview`).
- **Crash reporting** ‚Äî optional. Set `EXPO_PUBLIC_SENTRY_DSN`, install
  `@sentry/react-native`, add the Expo Sentry plugin, then re-run `prebuild`.
  Without a DSN the app no-ops (see `lib/sentry.ts` + root `AppErrorBoundary`).
- **Checks** ‚Äî `pnpm --filter @backsteros/mobile typecheck` and
  `pnpm --filter @backsteros/mobile test`.

## Foundation checklist ‚Äî Journal on iOS Simulator

1. **API** ‚Äî production `https://service.backsteros.com` or local
   `backsteros-api` on `http://127.0.0.1:8787` with Clerk and a workspace.
2. **Env** ‚Äî `backsteros-mobile/.env` with `EXPO_PUBLIC_CLERK_PUBLISHABLE_KEY` and
   `EXPO_PUBLIC_API_URL` pointing at that API.
3. **Seed** ‚Äî at least one journal document (create via web/desktop/API).
4. **Native build** (not Expo Go):

   ```bash
   pnpm --filter @backsteros/mobile prebuild
   pnpm --filter @backsteros/mobile ios
   ```

5. **Verify** ‚Äî Simulator opens ‚Üí sign in ‚Üí **Journal** lists entry titles.
   Tasks tab covers due-date filters (Today / Tomorrow / ‚Ä¶).

Physical device: point `EXPO_PUBLIC_API_URL` at your LAN IP instead of
`127.0.0.1`.
